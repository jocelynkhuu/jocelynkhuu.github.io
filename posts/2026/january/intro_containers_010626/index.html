What are containers?

Differences between VMs and containers

I wrote a post a couple of years ago on how to use Podman, but what exactly are containers and how do they differ from VMs?

Let’s start with VMs.

Virtual Machines (VM)

A VM is basically a virtual computer that can run operating systems within a virtualized environment on a single physical host machine. Hypervisors manage VMs and there are Type 1 and Type 2 Hypervisors. Type 1 is what typically runs in datacenters directly on hardware. Type 2 runs as an app on a host OS such as VirtualBox, VMware Fusion, etc.

Virtual machines are great for customization and security as they run their own kernel, own guest OS, and own libraries making them pretty isolated. Many VMs can run on one host for scalability but VMs tend to be more resource intensive than containers. Each VM has to be individually patched as they run their own OS and kernel.

Containers

Containers are isolated processes that run an application with its dependencies while sharing the host OS’ kernel. Kind of like a sandbox for a process. They scale well and are extremely lightweight and portable as they don’t require a guest OS, are often ephemeral, and all of the dependencies are packaged together to run on the host OS’ kernel. They do have their own security risks though as they share the Host OS kernel. Meaning that if vulnerabilities are present at the kernel level, all containers are vulnerable to it.

Containers have 3 basic components to run:

  1. A manifest such as a Dockerfile, which describes how the container image should be built. This acts as a blueprint for the application, including which base image and libraries are required and how the application is started.
  2. A container image that is created from the Dockerfile as an immutable snapshot of the application
  3. The container runtime which runs on the host OS and is responsible for isolation and managing the container resources. This is kind of like the hypervisor

A container can be destroyed and redeployed very quickly and container images can be updated and versioned in a container image repository.

The key difference between virtual machines and containers is that virtual machines isolate entire machines, while containers isolate processes.

Container runtime protection is very important to securing containers. Container runtime protection typically runs on the Host OS to monitor container processes in real time and detect anomalies or suspicious activity. Since containers are immutable, any unexepected behavior could indicate a compromise.
